### 3.2 Request DTO Interfaces

```typescript
// Base visitor registration fields (common to both MEETING and DELIVERY)
interface BaseSecurityRegistrationDto {
  phone: string;                    // Required: 10 digits, from PhoneLookupFlow
  branchId: string;                 // Required: UUID v4, matches authenticated security user
  firstName: string;                // Required: Min 2 characters
  lastName: string;                 // Required: Min 2 characters
  email?: string;                   // Optional: Valid email (required for MEETING)
  company?: string;                 // Optional: Visitor's organization
  designation?: string;             // Optional: Job title (required for MEETING)
  address?: string;                 // Optional: Visitor's address
}

// Meeting visit variant - extends base with meeting-specific fields
interface SecurityMeetingRegistrationDto extends BaseSecurityRegistrationDto {
  visitType: 'MEETING';            // Discriminator field
  email: string;                    // Required for MEETING visits
  designation: string;               // Required for MEETING visits
  department?: Department;          // Optional: Department enum
  hostSelectionMode: 'dropdown' | 'manual';  // Required: How host is selected
  hostId?: string;                 // If hostSelectionMode === 'dropdown'
  staffName?: string;               // If hostSelectionMode === 'manual'
  staffPhone?: string;              // If hostSelectionMode === 'manual'
  purpose: string;                  // Required: Min 5 characters
}

// Delivery visit variant - extends base with delivery-specific fields
interface SecurityDeliveryRegistrationDto extends BaseSecurityRegistrationDto {
  visitType: 'DELIVERY';           // Discriminator field
  deliveryPlatform: string;          // Required: Min 2 characters
  deliveryRecipient: string;         // Required: Min 2 characters
  orderReference?: string;           // Optional: Order/AWB number
}

// Discriminated union type for security registration
type SecurityRegistrationDto =
  | SecurityMeetingRegistrationDto
  | SecurityDeliveryRegistrationDto;
```

### 3.3 Response DTO Interfaces

```typescript
// Successful security registration response
interface SecurityRegistrationResponse {
  success: true;
  message: string;
  visitor: VisitorResponseData;
  visit: VisitResponseData;
  whatsappSent: boolean;

// Subset of visitor data returned in response
interface VisitorResponseData {
  id: string;
  firstName: string;
  lastName: string;
  phone: string;
  email?: string | null;
  company?: string | null;
  photo?: string | null;            // GCP URL of uploaded photo
}

// Visit data returned in response (Status is always CHECKED_IN for security-initiated registrations)
interface VisitResponseData {
  id: string;
  visitType: 'MEETING' | 'DELIVERY';
  status: 'CHECKED_IN';            // Always CHECKED_IN for this endpoint
  checkInTime: Date;
  checkInOtp: string;              // 6-digit OTP (or "654321" in TEST_MODE)
  checkInOtpExpiry: Date;
  gatePassUrl: string;             // GCP signed URL (7-day expiry)
  gatePassUrlExpiry: Date;
  department?: string | null;
  hostName?: string | null;
  hostPhone?: string | null;
  purpose?: string | null;
  deliveryPlatform?: string | null;
  deliveryRecipient?: string | null;
}
```

## 4. Validation Rules

| Field | DTO Type | Required | Validation Rules |
|-------|----------|----------|-----------------|
| **Base Fields** | | | |
| phone | All | Yes | Must be exactly 10 digits, numeric only |
| branchId | All | Yes | Must be valid UUID v4 format |
| firstName | All | Yes | Minimum 2 characters |
| lastName | All | Yes | Minimum 2 characters |
| email | All | Conditional | Required for MEETING, valid email format |
| company | All | No | No specific validation |
| designation | All | Conditional | Required for MEETING, minimum 2 characters |
| address | All | No | No specific validation |
| **Meeting Fields** | | | |
| visitType | Meeting | Yes | Must be 'MEETING' |
| department | Meeting | No | Must be valid Department enum value |
| hostSelectionMode | Meeting | Yes | Must be 'dropdown' or 'manual' |
| hostId | Meeting | Conditional | Required if hostSelectionMode === 'dropdown', UUID v4 |
| staffName | Meeting | Conditional | Required if hostSelectionMode === 'manual', min 2 chars |
| staffPhone | Meeting | Conditional | Required if hostSelectionMode === 'manual', 10 digits |
| purpose | Meeting | Yes | Minimum 5 characters |
| **Delivery Fields** | | | |
| visitType | Delivery | Yes | Must be 'DELIVERY' |
| deliveryPlatform | Delivery | Yes | Minimum 2 characters |
| deliveryRecipient | Delivery | Yes | Minimum 2 characters |
| orderReference | Delivery | No | No specific validation |
| **File Uploads** | | | |
| photo | All | Yes | JPG/PNG only, max 5MB |
| governmentIdDocument | All | No | JPG/PNG/PDF only, max 5MB |
| officeIdDocument | All | No | JPG/PNG/PDF only, max 5MB |

## 6. Multipart Request Structure

### 6.1 Request Format

The endpoint uses `multipart/form-data` encoding with:

1. **JSON Data Field:** `data` field containing the DTO as JSON string
2. **File Fields:** Separate file fields for photo and documents

### 6.2 FormData Structure


## 7. Acceptance Criteria

- [ ] Base DTO (`SecurityRegistrationBaseDto`) contains all common fields (phone, branchId, firstName, lastName, optional fields)
- [ ] Meeting DTO (`SecurityMeetingRegistrationDto`) extends base with meeting-specific fields (email, designation, host info, purpose)
- [ ] Delivery DTO (`SecurityDeliveryRegistrationDto`) extends base with delivery-specific fields (deliveryPlatform, deliveryRecipient, orderReference)
- [ ] All DTOs use `class-validator` decorators for input validation
- [ ] All DTOs use `@nestjs/swagger` decorators for API documentation
- [ ] Discriminated union on `visitType` field distinguishes MEETING vs DELIVERY variants
- [ ] Phone validation: exactly 10 digits, numeric only
- [ ] Branch ID validation: valid UUID v4 format
- [ ] Name validation: minimum 2 characters for first and last name
- [ ] Email validation: valid email format (required for MEETING visits)
- [ ] Host selection validation: conditional validation based on `hostSelectionMode` (dropdown requires `hostId`, manual requires `staffName` and `staffPhone`)
- [ ] Purpose validation: minimum 5 characters (MEETING visits)
- [ ] Delivery validation: minimum 2 characters for platform and recipient
- [ ] Response DTO (`SecurityRegistrationResponse`) includes visitor, visit, OTP, gate pass URL, and WhatsApp status
- [ ] Multipart request structure implemented with JSON `data` field + separate file fields
