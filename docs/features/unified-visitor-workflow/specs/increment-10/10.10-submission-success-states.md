## 4. Component Structure

```typescript
/**
 * SecurityRegistrationModal component with submission and success states
 * Conditional rendering based on current step
 */
function SecurityRegistrationModal(props: SecurityRegistrationModalProps): JSX.Element;
```

**Conditional Rendering Logic:**
- Step `visitType`: Render type selection cards
- Step `visitorForm`: Render meeting or delivery form
- Step `processing`: Render processing state with spinner
- Success state (`successResponse != null`): Render success confirmation

**State Transitions:**
- Form submit → `processing` → success/error
- Success actions: View Gate Pass, Register Another, Done

## 5. Logic Flow

### 5.1 Form Submission Handler

```typescript
async function handleSubmit(data: SubmissionData): Promise<void> {
  // Set processing state
  updateState({
    step: 'processing',
    isSubmitting: true,
    processingMessage: 'Registering visitor...',
    submissionError: null,
  });

  try {
    // Build FormData for multipart upload
    const formData = buildMultipartFormData(data);

    // Update processing message
    updateState({ processingMessage: 'Generating gate pass...' });

    // Call API
    const response = await registerSecurityVisitor(formData);

    // Update processing message
    updateState({ processingMessage: 'Sending via WhatsApp...' });

    // Small delay for UX (500ms)
    await new Promise((resolve) => setTimeout(resolve, 500));

    // Success state
    updateState({
      step: 'success',
      isSubmitting: false,
      processingMessage: 'Complete',
      successResponse: response,
      submissionError: null,
    });

    // Trigger parent callback (dashboard refresh)
    props.onRegistrationComplete?.();

  } catch (error) {
    // Error handling
    const errorMessage = parseSubmissionError(error);
    updateState({
      step: 'visitorForm', // Return to form
      isSubmitting: false,
      submissionError: errorMessage,
      processingMessage: 'Complete',
      successResponse: null,
    });
  }
}
```

### 5.2 Build Multipart FormData

```typescript
function buildMultipartFormData(data: SubmissionData): FormData {
  const formData = new FormData();

  // Append JSON data field
  const jsonData = { ...data.formData, branchId: data.branchId };
  formData.append('data', JSON.stringify(jsonData));

  // Append file fields
  formData.append('photo', data.files.photo);
  if (data.files.governmentIdDocument) {
    formData.append('governmentIdDocument', data.files.governmentIdDocument);
  }
  if (data.files.officeIdDocument) {
    formData.append('officeIdDocument', data.files.officeIdDocument);
  }

  return formData;
}
```

### 5.3 Processing State

Displays centered spinner with processing message. Shows progress indicators:
- "Registering visitor..." (checkmark when complete)
- "Generating gate pass..." (checkmark when complete)
- "Sending via WhatsApp..." (spinner while pending)
- Modal close prevented during processing

### 5.4 Success Confirmation

Displays:

- "Checked In" badge (emerald)
- WhatsApp status (green banner if sent, amber warning if failed)
- Three action buttons: "View Gate Pass", "Register Another Visitor", "Done"

### 5.5 Gate Pass View Overlay

Opens as full-screen overlay. Contains:
- Header with "Checked In" badge and close button
- GatePassView component with `checkedIn={true}`
- Footer with "Download" and "Close" buttons
- Download opens `response.visit.gatePassUrl` in new tab

## 6. Error Handling

### 6.1 Submission Errors

| Error Type | HTTP Status | Error Code | Display Message | Recovery |
|------------|-------------|------------|----------------|-----------|
| Network Error | N/A | `NETWORK_ERROR` | "Network error. Please check your connection and try again." | Retry button |
| Validation Failed | 400 | `VALIDATION_FAILED` | "Invalid form data. Please check and try again." | Retry after fixing |
| File Upload Error | 400 | `FILE_UPLOAD_ERROR` | "Failed to upload files. Please try again." | Retry with different files |
| GCP Upload Failed | 500 | `GCP_UPLOAD_FAILED` | "Failed to generate gate pass. Please try again." | Retry |
| WhatsApp Failed | 200 (warning) | `WHATSAPP_SEND_FAILED` | Warning banner (not error) | Manually share gate pass |
| Server Error | 500 | `SERVER_ERROR` | "Something went wrong. Please try again." | Retry |

### 6.2 Error Display & Recovery

- **Error Banner:** Red background, AlertCircle icon, dismissible with X button
- **Form Data Preservation:** Keep all form data when error occurs
- **Retry:** "Try Again" button allows re-submission without re-entering data
- **WhatsApp Failure:** Non-blocking warning, gate pass still displayed

## 11. Implementation Notes

### 11.1 Processing Message Timing

- Update processing message at specific points in submission flow
- Add small delay (500ms) between messages for visual feedback
- Use `setTimeout` for message transitions

### 11.2 FormData Structure

- Use standard `FormData` API
- Append JSON data as string field named "data"
- Append files with correct field names: "photo", "governmentIdDocument", "officeIdDocument"
- Backend expects this structure (from Task 10.1)

### 11.3 WhatsApp Non-Blocking

- WhatsApp failure does not prevent success confirmation
- Display warning banner but still show gate pass
- Allow manual sharing of gate pass

### 11.4 Gate Pass View Overlay

- Render as fixed overlay with z-index above modal
- Focus trap within overlay when open
- Return focus to "View Gate Pass" button on close

### 11.5 Parent Callbacks

- `onRegistrationComplete()` called after successful submission
- Parent component (dashboard) uses this to refresh visitor logs

### 11.6 Edge Cases

1. **Rapid Submissions:** Prevent multiple submissions with isSubmitting flag
2. **Unmount During Submission:** Cancel pending operations, cleanup state
3. **Slow Network:** Processing state provides visual feedback during wait
4. **WhatsApp Partial Failure:** Handle mixed success/failure scenarios
