## 5. Logic Flow

### 5.1 State Initialization

```typescript
function SecurityDashboardPage(): JSX.Element {
  const user = useAuthenticatedUser();
  const [registrationModal, setRegistrationModal] = useState<RegistrationModalState>({
    open: false,
    phone: '',
  });
  const [activeTab, setActiveTab] = useState<'checkin' | 'logs'>('checkin');
  const toast = useToast();
  const { refetch: refetchLogs } = useVisitorLogs(user.branchId);
}
```

### 5.2 Open Registration Modal Handler

```typescript
/**
 * Opens registration modal with pre-filled phone number
 * Called from PhoneLookupFlow when user clicks "Register as new visitor"
 */
const handleRegisterNew = useCallback((phone: string, branchId: string): void => {
  setRegistrationModal({ open: true, phone });
}, []);
```

### 5.3 Close Modal Handler

```typescript
const handleCloseModal = useCallback((): void => {
  setRegistrationModal({ open: false, phone: '' });
}, []);
```

### 5.4 Registration Complete Handler

```typescript
/**
 * Called when visitor registration completes successfully
 * Triggers logs refresh and shows toast notification
 */
const handleRegistrationComplete = useCallback((): void => {
  refetchLogs();
  toast({
    type: 'success',
    title: 'Visitor registered',
    message: 'Visitor registered and checked in successfully',
    duration: 3000,
  });
}, [refetchLogs, toast]);
```

### 5.5 Integration Points

```typescript
// PhoneLookupFlow with new callback
<PhoneLookupFlow
  branchId={user.branchId}
  onVisitorFound={handleVisitorFound}
  onRegisterNew={handleRegisterNew}
  onBack={handleBack}
/>

// SecurityRegistrationModal overlay
<SecurityRegistrationModal
  open={registrationModal.open}
  onClose={handleCloseModal}
  phone={registrationModal.phone}
  branchId={user.branchId}
  securityUserId={user.id}
  onRegistrationComplete={handleRegistrationComplete}
/>
```

### 5.6 Complete Flow Sequence

1. User enters phone in PhoneLookupFlow → API returns "not found"
2. User clicks "Register as new visitor" → handleRegisterNew() opens modal
3. SecurityRegistrationModal renders with pre-filled phone
4. User completes form (meeting or delivery) → submits
5. SecurityRegistrationModal processes submission → shows success
6. User clicks "Done" → onClose() + onRegistrationComplete()
7. handleRegistrationComplete() → refetchLogs() + toast.success()

## 7. Error Handling

| Scenario | Handling |
|----------|----------|
| Modal fails to open/close | Log error, show toast warning |
| refetchLogs() fails | Log error, show toast error |
| Empty phone number | Validate before opening modal |
| Modal already open | Ignore duplicate open calls |
| Network timeout during refresh | Show error, allow manual refresh |

## 11. Implementation Notes

- Use local state (`useState`) for modal - simple single-purpose state, no global context needed
- Memoize callbacks with `useCallback` to prevent unnecessary re-renders
- SecurityRegistrationModal handles all internal modal logic (focus trap, animations, form steps)
- Logs refresh happens in background via TanStack Query's `refetch()`
- Modal renders at root level of dashboard with z-50 to appear above all content
