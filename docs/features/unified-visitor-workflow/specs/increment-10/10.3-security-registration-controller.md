## 4. Function Signatures

### 4.1 Controller Method

```typescript
@Controller('security')
@UseGuards(JwtAuthGuard, RolesGuard)
export class SecurityController {
  constructor(
    private readonly securityRegistrationService: SecurityRegistrationService,
  ) {}

  @Post('register-visitor')
  @ApiOperation({ summary: 'Register visitor with security assistance' })
  @ApiConsumes('multipart/form-data')
  @ApiBody({ type: 'multipart/form-data' })
  @ApiResponse({ status: 200, description: 'Registration successful', type: SecurityRegistrationResponse })
  @ApiResponse({ status: 400, description: 'Validation error', type: ErrorResponse })
  @ApiResponse({ status: 401, description: 'Unauthorized' })
  @ApiResponse({ status: 403, description: 'Forbidden - Branch mismatch' })
  @ApiResponse({ status: 409, description: 'Visitor already exists', type: ErrorResponse })
  @ApiResponse({ status: 500, description: 'Internal server error' })
  @Roles('SECURITY', 'SECURITY_SUPERVISOR')
  @UseInterceptors(
    FileFieldsInterceptor([
      { name: 'governmentIdDocument', maxCount: 1 },
      { name: 'officeIdDocument', maxCount: 1 },
    ]),
  )
  async registerVisitor(
    @Body() data: any,
    @UploadedFiles() files: VisitorFiles,
    @Request() req: { user: JwtPayload },
  ): Promise<SecurityRegistrationResponse>;
}
```

**Decorator Summary:**
- `@Post('register-visitor')`: Defines POST endpoint at `/security/register-visitor`
- `@ApiOperation/@ApiConsumes/@ApiBody/@ApiResponse`: Swagger documentation for multipart/form-data with file uploads
- `@UseGuards(JwtAuthGuard, RolesGuard)`: Enforces JWT authentication and role validation
- `@Roles('SECURITY', 'SECURITY_SUPERVISOR')`: Restricts access to security roles only
- `@UseInterceptors(FileFieldsInterceptor)`: Handles optional document uploads (max 1 file per field)

## 5. Controller Logic (Pseudo-Code)

### 5.1 Main Flow: `registerVisitor()`

```typescript
async registerVisitor(
  data: any,
  files: VisitorFiles,
  req: { user: JwtPayload },
): Promise<SecurityRegistrationResponse> {
  // Step 1: Parse JSON data field from multipart body
  const dto: SecurityRegistrationDto = JSON.parse(data.data);

  // Step 2: Validate request structure
  if (!files || !files.photo) {
    throw new BadRequestException({
      code: 'PHOTO_REQUIRED',
      message: 'Visitor photo is required',
    });
  }

  // Step 3: Validate branchId matches authenticated user
  if (dto.branchId !== req.user.branchId) {
    throw new ForbiddenException({
      code: 'BRANCH_MISMATCH',
      message: 'Branch ID must match authenticated user branch',
    });
  }

  // Step 4: Validate DTO using class-validator
  const validationResult = await validate(dto);
  if (validationResult.length > 0) {
    throw new BadRequestException({
      code: 'VALIDATION_ERROR',
      message: 'Validation failed',
      details: validationResult.map(err => ({
        field: err.property,
        message: Object.values(err.constraints || {}).join(', '),
      })),
    });
  }

  // Step 5: Call service to handle registration logic
  const response = await this.securityRegistrationService.registerVisitor(
    dto,
    files,
    req.user,
  );

  // Step 6: Return response
  return response;
}
```

## 6. Middleware & Interceptor Configuration

### 6.1 File Upload Interceptor

The `FileFieldsInterceptor` handles optional document uploads:
- Configured for `governmentIdDocument` and `officeIdDocument` fields
- Each field accepts max 1 file
- Photo handled separately via request body parsing

### 6.2 Global Multer Configuration

File upload constraints (configured in `app.module.ts` or separate config):
- **Max file size:** 5MB per file
- **Allowed types:** `image/jpeg`, `image/png`, `application/pdf`
- **File filter:** Rejects unsupported MIME types with error response

### 6.3 Authentication & Authorization

Two guards applied in sequence:
1. **JwtAuthGuard:** Verifies JWT token validity and extracts `JwtPayload` from `req.user`
2. **RolesGuard:** Validates user has 'SECURITY' or 'SECURITY_SUPERVISOR' role

Branch validation ensures security user can only register visitors for their own branch (Step 3 in controller logic).

## 7. Error Handling (Reference Task 10.1)

### 7.1 Error Codes

| Error Code | HTTP Status | Scenario |
|------------|--------------|----------|
| `PHOTO_REQUIRED` | 400 | Photo file not provided in request |
| `VALIDATION_ERROR` | 400 | DTO validation failed (detailed in 10.1) |
| `INVALID_FILE_TYPE` | 400 | Uploaded file type not allowed |
| `BRANCH_MISMATCH` | 403 | Request branchId does not match user branchId |
| `FILE_UPLOAD_ERROR` | 400 | File upload failed (size, type, or network) |
| `GCP_UPLOAD_FAILED` | 500 | Failed to upload gate pass to GCP |
| `UNAUTHORIZED` | 401 | User not authenticated |
| `FORBIDDEN` | 403 | User lacks required role |

### 7.2 Error Response Format

interface ErrorResponse {

## 8. API Documentation Summary

### 8.1 Swagger Documentation

The endpoint is documented with:
- `@ApiOperation` summary describing security-assisted registration
- `@ApiConsumes('multipart/form-data')` indicating multipart request format
- `@ApiBody` schema describing `data` (JSON), `photo` (required), and optional document fields
- `@ApiResponse` entries for all success and error scenarios

### 8.2 Request Format

**Endpoint:** `POST /api/security/register-visitor`

**Headers:**
- `Authorization: Bearer <JWT_TOKEN>`

**Body (multipart/form-data):**
- `data`: JSON string containing `SecurityRegistrationDto`
- `photo`: File (required, JPG/PNG, max 5MB)
- `governmentIdDocument`: File (optional, JPG/PNG/PDF, max 5MB)
- `officeIdDocument`: File (optional, JPG/PNG/PDF, max 5MB)

**Response:** `SecurityRegistrationResponse` with visitor, visit, OTP, and gate pass URL
