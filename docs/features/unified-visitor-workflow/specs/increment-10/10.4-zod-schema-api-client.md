## 4. Zod Schema Pattern

### 4.1 Discriminated Union Structure

The schema uses a **discriminated union pattern** with `visitType` as the discriminator. This ensures type-safe validation based on visit type.

```typescript
import { z } from 'zod';

// Base schema with common fields
export const securityRegistrationBaseSchema = z.object({
  phone: z.string().length(10).regex(/^\d{10}$/),
  firstName: z.string().min(2).trim(),
  lastName: z.string().min(2).trim(),
  email: z.string().email().optional(),
  company: z.string().optional(),
  designation: z.string().optional(),
  address: z.string().optional(),
});

// Meeting schema (extends base, requires email/designation/host/purpose)
export const securityMeetingSchema = securityRegistrationBaseSchema.extend({
  visitType: z.literal('MEETING'),
  email: z.string().email(),
  designation: z.string().min(2),
  department: z.enum(['ADMINISTRATION', 'CARDIOLOGY', /* ... */]),
  hostSelectionMode: z.enum(['dropdown', 'manual']),
  hostId: z.string().uuid().optional(),
  staffName: z.string().min(2).optional(),
  staffPhone: z.string().regex(/^\d{10}$/).optional(),
  purpose: z.string().min(5),
}).refine(
  (data) => data.hostSelectionMode === 'dropdown' ? !!data.hostId : !!(data.staffName && data.staffPhone),
  { message: 'Host selection is required', path: ['hostId'] },
);

// Delivery schema (extends base, requires platform/recipient)
export const securityDeliverySchema = securityRegistrationBaseSchema.extend({
  visitType: z.literal('DELIVERY'),
  deliveryPlatform: z.string().min(2),
  deliveryRecipient: z.string().min(2),
  orderReference: z.string().optional(),
});

// Discriminated union - validates based on visitType field
export const securityRegistrationSchema = z.discriminatedUnion('visitType', [
  securityMeetingSchema,
  securityDeliverySchema,
]);

// Type inference
export type SecurityRegistrationFormData = z.infer<typeof securityRegistrationSchema>;
```

### 4.2 Validation Rules Reference

Detailed validation rules are defined in **Task 10.1 (Backend DTO)**. Key rules include:
- Phone: exactly 10 digits, numeric only
- Names: minimum 2 characters, trimmed
- Email: valid format (required for MEETING)
- Host selection: conditional based on `hostSelectionMode`
- Purpose: minimum 5 characters (MEETING only)
- Delivery fields: minimum 2 characters

## 5. API Client Function

### 5.1 Function Signature

```typescript
/**
 * API client function for security-assisted visitor registration
 *
 * @param request - Form data and files to submit
 * @returns Promise with registration response
 * @throws ApiError - On API errors with specific error codes
 */
export async function registerSecurityVisitor(
  request: RegisterSecurityVisitorRequest,
): Promise<SecurityRegistrationResponse>;
```

### 5.2 Implementation (Pseudo-Code)

```typescript
import apiClient from '@/lib/api';
import { ApiError } from '@/lib/api/visitors-api';

export async function registerSecurityVisitor(
  request: RegisterSecurityVisitorRequest,
): Promise<SecurityRegistrationResponse> {
  // Step 1: Create FormData instance
  const formData = new FormData();

  // Step 2: Append JSON data field
  formData.append('data', JSON.stringify(request.data));

  // Step 3: Append photo (required)
  formData.append('photo', request.files.photo);

  // Step 4: Append optional documents
  if (request.files.governmentIdDocument) {
    formData.append('governmentIdDocument', request.files.governmentIdDocument);
  }
  if (request.files.officeIdDocument) {
    formData.append('officeIdDocument', request.files.officeIdDocument);
  }

  // Step 5: Make API call
  try {
    const response = await apiClient.post<SecurityRegistrationResponse>(
      '/api/security/register-visitor',
      formData,
      { /* headers: let browser set Content-Type to multipart/form-data */ }
    );
    return response.data;
  } catch (error) {
    // Step 6: Handle API errors
    if (axios.isAxiosError(error) && error.response) {
      const { statusCode, code, message } = error.response.data;
      throw new ApiError({ statusCode, code, message });
    }
    throw new ApiError({
      statusCode: 0,
      code: 'NETWORK_ERROR',
      message: 'Network error. Please check your connection.',
    });
  }
}
```

## 6. Error Handling

### 6.1 API Error Codes

| Error Code | HTTP Status | Frontend Action |
|------------|--------------|-----------------|
| `PHOTO_REQUIRED` | 400 | Show "Visitor photo is required" |
| `VALIDATION_ERROR` | 400 | Display field-specific errors from details array |
| `INVALID_FILE_TYPE` | 400 | Show "Only JPG, PNG, and PDF files are allowed" |
| `BRANCH_MISMATCH` | 403 | Show "Branch ID must match authenticated user branch" |
| `FILE_UPLOAD_ERROR` | 400 | Show "Failed to upload files. Please try again." |
| `GCP_UPLOAD_FAILED` | 500 | Show "Failed to generate gate pass. Please try again." |
| `NETWORK_ERROR` | 0 | Show "Network error. Please check your connection." |

### 6.2 Error Response Format

```typescript
interface ErrorResponse {
  success: false;
  code: string;
  message: string;
  details?: Array<{ field: string; message: string }>;
}
```

### 6.3 Error Handling Pattern

```typescript
try {
  const response = await registerSecurityVisitor({ data, files });
} catch (error) {
  if (error instanceof ApiError) {
    switch (error.code) {
      case 'PHOTO_REQUIRED':
        setError('photo', { message: 'Visitor photo is required' });
        break;
      case 'VALIDATION_ERROR':
        error.details?.forEach(detail => setError(detail.field, { message: detail.message }));
        break;
      default:
        showToast('error', error.message);
    }
  }
}
```
