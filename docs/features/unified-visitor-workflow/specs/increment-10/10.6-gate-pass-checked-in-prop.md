## 1. Overview

Add optional `checkedIn` prop to the existing `GatePassView` component to display "Checked In" badge for security-initiated registrations. When `checkedIn` is `true`, the component shows a green "Checked In" badge instead of "Approved", and removes the validity expiry warning.

### Key Requirements
- Add optional `checkedIn?: boolean` prop
- Display "Checked In" badge (emerald) when `checkedIn={true}`
- Hide validity expiry warning when `checkedIn={true}`
- Maintain backward compatibility

## 3. Data Models

### 3.1 Component Props Interface (Updated)

```typescript
export interface GatePassViewProps {
  /** Complete visitor data object */
  visitor: GatePassVisitorData;

  /** 6-digit Check-In OTP */
  otp: string;

  /** Timestamp when gate pass expires */
  validityTimestamp: Date;

  /** Display skeleton loader instead of content */
  loading?: boolean;

  /** Error message to display (takes priority over expired) */
  error?: string | null;

  /** Mark gate pass as expired (muted styling) */
  expired?: boolean;

  /** Show QR code placeholder (future enhancement) */
  showQRCode?: boolean;

  /** NEW: Display "Checked In" badge instead of "Approved" */
  checkedIn?: boolean;

  /** Additional CSS classes for container */
  className?: string;
}
```

## 4. Component Changes

### 4.1 Function Signature Update

```typescript
export function GatePassView({
  visitor,
  otp,
  validityTimestamp,
  loading = false,
  error = null,
  expired = false,
  showQRCode = false,
  checkedIn = false,  // NEW prop with default false
  className,
}: GatePassViewProps): JSX.Element;
```

### 4.2 New Logic

```typescript
// Badge text and styling (same styling for both states)
const badgeLabel = checkedIn ? 'Checked In' : 'Approved';

// Validity section: hide for checked-in passes
const showValiditySection = !checkedIn && state === 'success';
```

### 4.3 JSX Updates

```typescript
// Badge rendering (same styling, different text)
<Badge className="bg-emerald-500 hover:bg-emerald-600 text-white border-emerald-500">
  <CheckCircle className="h-3 w-3 mr-1" />
  {badgeLabel}
</Badge>

// Validity section (conditionally rendered)
{showValiditySection && (
  <div className="border-t pt-4">
    <div className="flex items-center justify-center gap-2 text-sm text-muted-foreground">
      <Clock className="h-4 w-4" />
      <span>Valid until: {formatValidityTime(validityTimestamp)}</span>
    </div>
  </div>
)}
```

## 6. Testing

### 6.1 Test Scenarios

Add the following test cases to `GatePassView.spec.ts`:

1. **Checked In Badge Display**
   - Given `checkedIn={true}`, when component renders, then display "Checked In" badge with emerald styling

2. **Approved Badge Default Behavior**
   - Given `checkedIn={false}` or omitted, when component renders, then display "Approved" badge with emerald styling

3. **Validity Section Hidden for Checked In**
   - Given `checkedIn={true}` and valid timestamp, when component renders, then validity section should not exist in DOM

4. **Validity Section Shown for Approved**
   - Given `checkedIn={false}` and valid timestamp, when component renders, then display "Valid until: [timestamp]" with Clock icon

5. **Backward Compatibility**
   - Given existing usage without `checkedIn` prop, when component renders, then maintain "Approved" badge behavior

6. **Checked In with Expiry State**
   - Given `checkedIn={true}` and expired timestamp, when component renders, then show expired state with "Checked In" badge

7. **Checked In with Error State**
   - Given `checkedIn={true}` and error message, when component renders, then show error state with message (badge not shown)
