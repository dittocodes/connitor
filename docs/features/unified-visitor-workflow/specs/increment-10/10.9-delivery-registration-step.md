## 6. Validation Rules

**See Task 10.1 for validation rules, Task 10.4 for complete Zod schema**

| Field | Validation |
|-------|------------|
| `firstName` | Required, min 2 chars |
| `lastName` | Required, min 2 chars |
| `phone` | Required, read-only (pre-filled) |
| `deliveryPlatform` | Required, min 2, max 100 chars |
| `deliveryRecipient` | Required, min 2, max 100 chars |
| `orderReference` | Optional, max 50 chars |
| `photo` | Required, max 5MB, JPEG/PNG |

**Custom Platform:** If `deliveryPlatform === 'Other'`, custom input required (min 2 chars).

**Timing:** On blur, on submit, live error display.

## 7. Error Handling

| Scenario | Error Message |
|----------|---------------|
| Invalid file type | "Only JPEG and PNG images are allowed" |
| File too large | "Photo must be less than 5MB" |
| Camera denied | "Camera access denied. Please allow camera access" |
| File read error | "Failed to read photo file" |

**Form Validation:** Inline errors below fields, red border on invalid fields, focus first invalid field on submit.

**Submission Errors:** Network errors handled by parent component (error banner, preserve data).

## 8. Styling Requirements

**See UX Design Document for detailed styling specifications**

**Layout:** Scrollable modal (~85% viewport height), sections: Personal Information, Visitor Photo, Delivery Details.

**Phone Field:** Gray background, read-only attribute.

**Photo Section:** 120x120px capture button with dashed border, camera icon, thumbnail preview with remove button.

**Platform:** Dropdown with pre-defined options, custom text input when "Other" selected.

**Buttons:** Back (ghost, "← Back"), Submit (primary, full-width, "Register & Check In", emerald-600).

**States:** Focus (emerald-500 border, focus ring), Disabled (gray-300), Loading (spinner, "Registering...").

## 9. Accessibility

**ARIA:** `role="form"`, `aria-label` on all inputs/buttons, `aria-required` on required fields, `aria-invalid` on invalid fields, `aria-describedby` linking errors, `aria-live` for announcements.

**Keyboard Navigation:** Tab through fields (First Name → Last Name → Photo → Platform → Recipient → Back → Submit), Enter to submit, Escape for back confirmation.

**Focus:** Auto-focus First Name on mount, focus first invalid field after validation failure.

**Touch Targets:** Min 44x44px (photo capture: 120x120px).

## 12. Implementation Notes

**Reuse Strategy:** `DeliveryRegistrationForm` modified with props for phone pre-fill and read-only mode. `DeliveryDetailsStep` used as-is.

**File Upload:** Standard HTML file input (`accept="image/jpeg,image/png"`), `URL.createObjectURL()` for preview, revoke on unmount.

**Platform Selection:** Standard `<select>` or custom dropdown, conditional custom input: `{isOtherPlatform && <input type="text" />}`.

**Form State:** `react-hook-form` for validation, local state for photo file/preview.

**Edge Cases:** No photo validation, invalid type/size, "Other" empty custom, rapid uploads, unmount during upload, back with unsaved changes, network error recovery.
