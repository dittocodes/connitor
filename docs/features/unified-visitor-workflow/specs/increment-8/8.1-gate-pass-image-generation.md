```typescript
async generateGatePassImage(
  visitId: string
): Promise<GatePassImageResponse> {
  // 1. Retrieve and prepare data
  const data = await this.prepareGatePassData(visitId);

  // 2. Download visitor photo if available
  let visitorPhoto: Buffer | null = null;
  if (data.visitorPhoto) {
    try {
      visitorPhoto = await this.gcpStorageService.downloadFile(
        data.visitorPhoto
      );
    } catch (error) {
      this.logger.warn(
        `Failed to download visitor photo for ${visitId}, using initials`
      );
      // Fallback to initials - do not throw
    }
  }

  // 3. Generate image
  const imageUrl = await this.renderGatePassImage(data, visitorPhoto);
```

### 5.2 Helper: prepareGatePassData

```typescript
private async prepareGatePassData(
  visitId: string
): Promise<GatePassImageData> {
  // 1. Query visit with relations
  const visit = await this.prisma.visit.findUnique({
    where: { id: visitId },
    include: {
      visitor: true,
      staff: true,
      branch: true,
    },
  });

  // 2. Validate visit exists
  if (!visit) {
    throw new NotFoundException(GatePassError.VISIT_NOT_FOUND);
  }

  // 3. Format visitor name
  const visitorName = `${visit.visitor.firstName}${
    visit.visitor.middleName ? ` ${visit.visitor.middleName}` : ''
  } ${visit.visitor.lastName}`.trim();

  // 4. Build response
  return {
    visitorName,
    visitorPhone: visit.visitor.phone,
    visitorPhoto: visit.visitor.photo,
    visitCategory: visit.visitCategory,
    checkInOtp: visit.checkInOtp || 'N/A',
    checkInOtpExpiry: visit.checkInOtpExpiry || new Date(),
    hostName: visit.staff?.name || visit.staffName,
    department: visit.staff?.department || visit.department,
    deliveryPlatform: visit.deliveryPlatform,
    deliveryRecipient: visit.deliveryRecipient,
    orderReference: visit.orderReference,
    branchName: visit.branch.name,
  };
}
```

```typescript
  data: GatePassImageData,

  // 1. Initialize canvas with portrait dimensions (mobile-friendly)
  const width = 400;
  const height = 600;
  const canvas = createCanvas(width, height);

  // 2. Draw white background

  //      - Load photo buffer as image
  //      - Create circular mask at center
  //      - Draw photo within circle
  //      - Draw accent color border around circle
  //    If photo loading fails or not available:
  //      - Extract initials from visitor name (first name initial + last name initial)
  //      - Draw circular background with gray fill and accent border
  //      - Draw white initials centered in circle

  //    - Visitor name (centered, bold, large font)

  //      - Label: "Meeting:" (bold)

  //    If visitCategory == DELIVERY:
  //      - Label: "Delivery:" (bold)
  //      - Platform name (regular font)
  //      - Recipient name (smaller, secondary color) if available

  //    - Gray background box with padding
  //    - Label: "CHECK-IN CODE" (bold, centered above OTP)
  //    - OTP value (very large, bold, monospace font, accent color)
```

## 8. Error Handling

### 8.1 Error Codes

```typescript
export enum GatePassError {
  VISIT_NOT_FOUND = 'VISIT_NOT_FOUND',
  GATE_PASS_GENERATION_FAILED = 'GATE_PASS_GENERATION_FAILED',
  PHOTO_DOWNLOAD_FAILED = 'PHOTO_DOWNLOAD_FAILED',
}
```

### 8.2 Error Scenarios

#### Scenario 1: Visit Not Found

**When:** Database query returns null for visitId

**Action:**
- Throw `NotFoundException` with code `VISIT_NOT_FOUND`
- Include message: "Visit with ID {visitId} not found"

**HTTP Status:** 404 Not Found

---

#### Scenario 2: Photo Download Failure (Graceful Fallback)

**When:** GCP storage service throws error during photo download

**Action:**
- Log warning: "Failed to download visitor photo for {visitId}, using initials"
- DO NOT throw error
- Continue with initials fallback

**Reasoning:** Gate pass should still be usable even without photo

---

#### Scenario 3: Photo Load Failure (Graceful Fallback)

**When:** Canvas `loadImage()` throws error when processing photo buffer

**Action:**
- Log error: "Failed to load visitor photo for {visitId}, using initials"
- DO NOT throw error
- Continue with initials fallback

**Reasoning:** Corrupted or invalid photo buffer should not prevent gate pass generation

---

#### Scenario 4: Canvas Rendering Failure

**When:** Canvas operations throw error (e.g., `toDataURL()` fails)

**Action:**
- Log error: "Canvas rendering failed for {visitId}" with stack trace
- Throw Error with code `GATE_PASS_GENERATION_FAILED`
- Include message: "Failed to generate gate pass image"

**HTTP Status:** 500 Internal Server Error

**Reasoning:** Complete rendering failure is a critical error that must be surfaced

---

#### Scenario 5: Invalid Initials Generation

**When:** Visitor name is empty or contains non-alphabetic characters

**Action:**
- Handle gracefully - use placeholder initials (e.g., "??" or "??")
- Log warning: "Could not extract initials for visit {visitId}"

**Reasoning:** Edge case should not break generation

---

#### Scenario 6: Missing Required Fields

**When:** Required database fields are null or undefined

**Action:**
- For visitorName: Use "Unknown Visitor"
- For visitorPhone: Use "N/A"
- For checkInOtp: Use "N/A"
- For checkInOtpExpiry: Use current timestamp

**Reasoning:** Display with fallback values rather than failing

## 11. Performance Considerations

- **Caching:** Consider caching generated images if generation is expensive
- **Photo Download:** Download visitor photo only once per image generation
- **Canvas Operations:** Keep canvas operations synchronous for predictability
- **Base64 Size:** 400x600px PNG typically 50-200KB base64 (suitable for WhatsApp)

## 12. Security Considerations

- **Access Control:** This is an internal service method; protect caller if exposed via endpoint
- **Visitor Photo:** Do not expose sensitive government ID photos (only visitor.profile photo)
- **OTP Visibility:** OTP is intentionally prominent (this is the gate pass purpose)
- **Data Privacy:** Gate pass contains visitor name and phone (acceptable for access document)
