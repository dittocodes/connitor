# Technical Specification: GET /public/visits/:visitId/gate-pass

> **Feature**: Unified Visitor Workflow
> **Increment**: 8 - Gate Pass Generation & Delivery
> **Task**: 8.4 - Create `GET /public/visits/:visitId/gate-pass` endpoint
> **Status**: Approved (Reviewed: 2026-02-12)

---

## 1. Overview

This endpoint provides a public, no-authentication method for visitors to retrieve their Gate Pass information using their visit UUID. The endpoint returns:

- Gate Pass public URL (GCP URL from Task 8.2)
- Check-In OTP (6-digit code)
- Validity timestamp (checkInOtpExpiry)
- URL expiration timestamp (gatePassUrlExpiry)
- Visitor name and phone (for display)
- Visit category and status

This endpoint is designed to be accessed directly from a link shared with the visitor (via SMS/WhatsApp) or embedded in the status check page. It only returns data for approved visits that have a gate pass generated.

**Auto-Regeneration**: If the stored gate pass URL has expired (based on `gatePassUrlExpiry`), the endpoint automatically regenerates the URL by:
1. Generating a new gate pass image
2. Uploading to GCP and obtaining a new signed URL
3. Updating the database with the new URL and expiry

This regeneration is completely transparent to the visitor and ensures visitors never see expired URLs.

**Security Consideration**: Relies on the cryptographic entropy of UUID v4 as the access token. Visit IDs are not enumerable. Future Task 9.6 will add 24-hour link validity.

## 2. Endpoint Definition

### 2.1 HTTP Method & Path

```
GET /public/visits/:visitId/gate-pass
```

### 2.2 Authentication

**None required.** This is a public endpoint.

**Access Control**: Access is controlled through UUID-based identification. Only visitors with the correct visit UUID can retrieve their gate pass.

### 2.3 Request Parameters

| Parameter | Type   | Location | Description                     | Validation                |
|-----------|--------|----------|---------------------------------|---------------------------|
| `visitId` | UUID   | Path     | The unique identifier of visit  | Must be valid UUID v4 format |

### 2.4 Request Headers

| Header          | Value                     | Description                          |
|-----------------|---------------------------|--------------------------------------|
| `Content-Type`  | `application/json`        | Accept header (optional)             |
| `User-Agent`    | Browser/App identifier    | Optional for analytics               |

| Status | Scenario                              |
|--------|---------------------------------------|

| `404`  | Visit not found, not approved, or no gate pass |

### 5.2 Service Flow

```typescript
async getGatePassPublic(visitId: string): Promise<GatePassResponse> {
  // 1. Query visit from database with required relations
  const visit = await this.prisma.visit.findUnique({
    where: { id: visitId },
    include: {
      visitor: true,  // Get visitor data
      branch: true,   // Get branch info
    },
  });

  // 2. Handle not found
  if (!visit) {
    throw new NotFoundException({
      success: false,
      error: {
        code: 'GATE_PASS_NOT_FOUND',
        message: 'Gate pass not found. Please contact security if you believe this is an error.',
        details: {
          visitId,
          reason: 'VISIT_NOT_FOUND',
        },
      },
    });
  }

  // 3. Validate visit status - only APPROVED or CHECKED_IN allowed
  if (visit.status !== VisitStatus.APPROVED && visit.status !== VisitStatus.CHECKED_IN) {
    throw new NotFoundException({
      success: false,
      error: {
        code: 'GATE_PASS_NOT_FOUND',
        message: 'Gate pass not found. Please contact security if you believe this is an error.',
        details: {
          visitId,
          reason: 'NOT_APPROVED',
        },
      },
    });
  }

  // 4. Validate gate pass data exists
  if (!visit.checkInOtp || !visit.checkInOtpExpiry || !visit.visitQRCode || !visit.gatePassGeneratedAt) {
    throw new NotFoundException({
      success: false,
      error: {
        code: 'GATE_PASS_NOT_FOUND',
        message: 'Gate pass not found. Please contact security if you believe this is an error.',
        details: {
          visitId,
          reason: 'NO_GATE_PASS',
        },
      },
    });
  }

  // 5. Future: Check 24-hour expiry (Task 9.6)
  // const now = new Date();
  // const visitDate = new Date(visit.createdAt);
  // const hoursSinceCreation = (now.getTime() - visitDate.getTime()) / (1000 * 60 * 60);
  // if (hoursSinceCreation > 24) {
  //   throw new HttpException(
  //     {
  //       success: false,
  //       error: {
  //         code: 'LINK_EXPIRED',
  //         message: 'This gate pass link has expired (24-hour validity). Please contact security for assistance.',
  //         details: { expiryDate: visitDate.toISOString(), visitId },
  //       },
  //     },
  //     410,
  //   );
  // }

  // 6. CHECK URL EXPIRATION AND REGENERATE IF NEEDED
  const now = new Date();
  let gatePassUrl = visit.visitQRCode;
  let gatePassUrlExpiry = visit.gatePassUrlExpiry;

  if (visit.gatePassUrlExpiry && visit.gatePassUrlExpiry < now) {
    // URL expired - trigger regeneration flow
    try {
      // 6a. Generate new gate pass image
      const { imageUrl: newImageBase64 } = await this.generateGatePassImage(visitId);

      // 6b. Upload to GCP and get new signed URL
      const { publicUrl: newUrl, urlExpiry: newExpiry } = await this.uploadGatePassToGcp(
        visitId,
        newImageBase64,
      );

      // 6c. Update database with new URL and expiry
      await this.prisma.visit.update({
        where: { id: visitId },
        data: {
          visitQRCode: newUrl,
          gatePassUrlExpiry: newExpiry,
        },
      });

      gatePassUrl = newUrl;
      gatePassUrlExpiry = newExpiry;

      this.logger.log(`Regenerated gate pass URL for visit ${visitId} (previous URL expired)`);
    } catch (error) {
      this.logger.error(`Failed to regenerate gate pass URL for visit ${visitId}: ${error.message}`);
      throw new InternalServerErrorException({
        success: false,
        error: {
          code: 'URL_REGENERATION_FAILED',
          message: 'Failed to retrieve gate pass. Please contact security.',
          details: {
            visitId,
            reason: 'GCP_ERROR',
          },
        },
      });
    }
  }

  // 7. Build response
  const responseData = {
    visitId: visit.id,
    status: visit.status as VisitStatus,
    visitor: {
      id: visit.visitor.id,
      firstName: visit.visitor.firstName,
      lastName: visit.visitor.lastName,
      fullName: `${visit.visitor.firstName} ${visit.visitor.lastName}`,
      phone: this.maskPhone(visit.visitor.phone),
    },
    visitCategory: visit.visitCategory as VisitCategory | null,
    gatePass: {
      checkInOtp: visit.checkInOtp,
      validUntil: visit.checkInOtpExpiry.toISOString(),
      gatePassUrl: gatePassUrl,
      urlExpiry: gatePassUrlExpiry ? gatePassUrlExpiry.toISOString() : '',
      generatedAt: visit.gatePassGeneratedAt.toISOString(),
      sentViaWhatsApp: visit.gatePassSentViaWhatsApp,
    },
    branch: {
      id: visit.branch.id,
      name: visit.branch.name,
    },
  };

  return {
    success: true,
    data: responseData,
  };
}
```

### 5.3 Helper Methods

```typescript
private isValidUUID(uuid: string): boolean {
  const uuidRegex = /^[0-9a-f]{8}-[0-9a-f]{4}-4[0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$/i;
  return uuidRegex.test(uuid);
}

private maskPhone(phone: string): string {
  // India format: +91 XXXXX XXXXX -> +91 999** 9999
  if (phone.startsWith('+91')) {
    const parts = phone.split(' ');
    if (parts.length >= 3) {
      const masked = parts[1].substring(0, 3) + '**';
      return `${parts[0]} ${masked} ${parts[2]}`;
    }
  }
  // Generic: mask middle 4 digits for 10-digit phone
  if (phone.length === 10) {
    return phone.substring(0, 3) + '****' + phone.substring(7);
  }
  // Fallback for other formats
  return phone.substring(0, 2) + '***' + phone.substring(phone.length - 2);
}
```

| Valid URL (not expired)     | ~50ms                  | Single database query + URL return |
| Expired URL (regeneration)  | ~400ms                 | Image generation + GCP upload + DB update |
| Multiple concurrent requests| ~50-400ms             | Depends on URL freshness        |

### 10.2 Optimization Notes

- Valid URLs return quickly (no regeneration overhead)
- Regeneration only occurs when URL is expired (7-day default)
- Database is updated with new URL, so subsequent requests return faster
- Regeneration failure is handled gracefully with 500 error

- India format: `+91 999** 9999`
- Generic format: `999****9999`

### 11.3 GCP URL Handling

- The `visitQRCode` field stores the full signed GCP URL (from Task 8.2)
- Return this URL directly without modification
- Ensure the URL is publicly accessible (no authentication required)
- URL expiry is controlled by GCP signed URL mechanism

### 11.4 URL Regeneration Process

The regeneration flow reuses existing methods from Task 8.2:

1. `generateGatePassImage(visitId)`: Generates new gate pass image (base64)
2. `uploadGatePassToGcp(visitId, imageBase64)`: Uploads to GCP, returns signed URL and expiry

These methods should be extracted from Task 8.2 for reuse.

### 11.5 Security Considerations

- **No authentication**: Relies solely on UUID entropy
- **No PII leakage**: Exclude sensitive visitor data
- **CORS**: Configure appropriate CORS headers for public access
- **Information hiding**: Return 404 for all failure cases (don't leak existence of unapproved visits)
- **Logging**: Log all access attempts for security auditing
- **Regeneration security**: Only regenerate for approved visits (validated before regeneration)

### 11.6 Frontend Integration

The frontend gate pass display will:
1. Display the gate pass image from `gatePassUrl`

3. Display validity timestamp in user-friendly format
4. Show visitor name and branch information
5. Handle error states gracefully (404, 410, 500, network errors)
6. Allow downloading/saving the gate pass image
7. Display URL expiry information (optional, for transparency)
