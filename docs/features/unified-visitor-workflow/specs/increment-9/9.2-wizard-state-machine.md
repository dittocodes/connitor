## 4. State Machine Architecture

### 4.1 Step Transition Rules

**Forward Path:** PHONE_ENTRY → PHONE_VERIFICATION → VISIT_TYPE_SELECTION → VISITOR_REGISTRATION → VISIT_DETAILS → CONFIRMATION

**Conditional Routing:**
- Delivery path: VISITOR_REGISTRATION shows `DeliveryRegistrationForm`, VISIT_DETAILS shows `DeliveryDetailsStep`
- Meeting path: VISITOR_REGISTRATION shows `MeetingRegistrationForm`, VISIT_DETAILS shows `MeetingDetailsStep`
- Already registered visitor: Skip VISITOR_REGISTRATION, jump to VISIT_DETAILS with pre-filled data

**Back Navigation:** All steps except PHONE_ENTRY and CONFIRMATION support back navigation

### 4.2 Browser History Management

- Step transitions: Do NOT push to browser history (internal navigation)
- Exit points: Push to history only when leaving wizard (Cancel, Done)
- Browser back: Intercept with confirmation dialog ("Cancel registration? Progress will be lost.")

### 4.3 Form Data Persistence

All form data stored in React state, never lost on back navigation. Phone data persists from Step 1a through all steps. Visitor data and visit details persist until final submission.

## 7. Navigation Flow

### 7.1 Forward Navigation

Validate step data → Update wizard state → Transition to next step → Scroll to top → Focus step header for accessibility

### 7.2 Back Navigation

Steps 1b-4 support back button. Previous step data restored from state, scroll to top, focus step header.

### 7.3 Cancel Flow

Cancel button resets wizard and redirects to landing page. If not on first step, show confirmation dialog ("Cancel registration? Progress will be lost.")

### 7.4 Browser Back Interception

Popstate event listener prevents default back navigation on non-first steps, shows cancel confirmation dialog. On first step, allows normal back navigation.

### 7.5 Step Animations

Fade in/out transitions (300ms) between steps. Optional slide up effect for mobile. Respects `prefers-reduced-motion`.

---

## 8. Error Handling

### 8.1 Network Errors

Preserve all form data in React state. Display inline error with retry button. User retries without re-entering data.

### 8.2 Validation Errors

Prevent forward transition. Display inline errors below invalid fields. Highlight fields with red border. Focus first invalid field.

### 8.3 API Errors

**400 Bad Request:** Invalid input data, inline error message
**429 Too Many Requests:** "Too many requests. Try again in X seconds" with retry-after
**500 Internal Server Error:** "Server error. Try again later"

### 8.4 Already Registered Visitor

Pre-fill registration form with existing visitor data. Skip VISITOR_REGISTRATION step, jump to VISIT_DETAILS. Display "Welcome back! Information pre-filled."

## 9. Accessibility

### 9.1 Focus Management

- Step transitions: Move focus to step heading or first input
- Modals: Trap focus within modal
