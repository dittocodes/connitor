# Task 9.4: Forms Integration

## 1. Task Description

Wire the visit type selection and registration form components into the unified visitor registration wizard. This task connects `VisitTypeSelection`, `DeliveryRegistrationForm`, `MeetingRegistrationForm`, `DeliveryDetailsStep`, and `MeetingDetailsStep` components into a coherent workflow that supports two distinct paths:
- **Delivery Path (Fast Track)**: Minimal fields (first name, last name, phone, photo) → platform/recipient details
- **Meeting Path (Full Registration)**: Complete fields (name, email, company, designation, address, photo, government ID, office ID) → department/host/purpose details

The integration must support back navigation with form data persistence, pre-fill existing visitor data, and manage conditional routing based on the selected visit type.

## 2. Component/Location

### Primary Files
- **Wizard Page**: `frontend/src/app/visitor-registration/wizard/page.tsx`
- **Step Components**: `frontend/src/components/visitors/steps/`

### 3.3 Validation Rules

**DeliveryRegistrationForm**: firstName (required, 2-50 chars), lastName (required, 2-50 chars), photo (required, max 5MB, JPEG/PNG only)

**MeetingRegistrationForm**: firstName (required, 2-50 chars), lastName (required, 2-50 chars), email (required, valid format), company (optional), designation (optional), address (optional), phone (required, verified), photo (required, max 5MB, JPEG/PNG), governmentIdDocument (required, max 5MB, JPEG/PNG/PDF), officeIdDocument (optional, max 5MB, JPEG/PNG/PDF)

**DeliveryDetailsStep**: platform (required, 2-100 chars), recipient (required, 2-100 chars)

**MeetingDetailsStep**: department (required, select from predefined), hostId (required, from staff search), purpose (required, 5-500 chars)

### 3.4 File Upload Handling
- **Photo Files**: Max 5MB, JPEG/PNG only; Display thumbnail preview; Allow clear and re-select; Inline error messages for invalid size/type; Revoke object URLs on unmount
- **Government ID (Meeting)**: Max 5MB, JPEG/PNG/PDF only; Same handling as photo
- **Office ID (Meeting, optional)**: Max 5MB, JPEG/PNG/PDF only; Same handling as photo

## 4. API Integration

### GET /api/public/staff/search

**Purpose**: Search staff members by name for meeting host selection

**Request Parameters**:
```typescript
interface StaffSearchParams {
  branchId: string;    // Required - Branch UUID
  query: string;       // Required - Search query (min 2 chars)
  department?: string; // Optional - Filter by department
}
```

**Success Response** (200):
```typescript
interface StaffLookupResponse {
  staff: StaffMember[];
  total: number;
}
```

**Error Responses**: 400 Bad Request (Missing/invalid branchId), 500 Internal Server Error (Backend fetch failed)

**When to Call**: User types minimum 2 characters; Query debounced by 400ms; Department filter changes

**Debounce Strategy**: 400ms delay; Cancel previous pending requests; Only trigger when query.length >= 2

**Error Handling**: Display "Unable to search hosts. Please try again."; Allow user to retry; Maintain existing selection on error

## 6. State Management

### Data Storage by Path

**Delivery Path State**:
```typescript
deliveryFormData: { firstName, lastName, photo } | null;
deliveryDetailsData: { platform, recipient } | null;
```

**Meeting Path State**:
```typescript
meetingFormData: {
  firstName, lastName, email, company?, designation?, address?,
  phone, photo, governmentIdDocument, officeIdDocument?
} | null;
meetingDetailsData: { department, hostId, purpose } | null;
selectedHost: StaffMember | null;
```

### Visit Type State Management
Set by `VisitTypeSelection.onSelect`; Stored in `wizardState.visitType`; Determines which form path to render; Cannot be changed after registration form step (must restart wizard)

### Form Data Persistence

**Back Navigation**: All form data stored in wizard state; Pass `initial*` props to pre-fill on back; Form dirty state preserved

**Forward Navigation**: Form data validated before transition; Only proceed if valid; Store data in appropriate wizard state field

**Browser Back/Forward**: Maintain sync with browser history; Restore wizard state on browser navigation; Optional: Preserve across page refresh (localStorage)

### Pre-fill Logic for Existing Visitors
```typescript
const getPrefillData = () => {
  if (isExistingVisitor && existingVisitorData) {
    return {
      firstName: existingVisitorData.firstName,
      lastName: existingVisitorData.lastName,
      email: existingVisitorData.email || '',
      company: existingVisitorData.company || '',
      designation: existingVisitorData.designation || '',
      address: existingVisitorData.address || '',
    };
  }
  return {};
};
```

## 8. Error Handling

### Network Error Handling
Preserve form data in wizard state; Display "Connection lost. Please check your internet and try again."; Show "Retry" button; Disable form fields during loading/error state

```typescript
const handleApiError = (error: Error) => {
  setSubmissionError(error.message);
  setSubmitting(false);
};
```

### Validation Error Handling

**Form Validation**: React Hook Form handles inline validation; Display error below each field; Prevent submission if invalid; Highlight invalid fields with red border

**File Upload Errors**: Size error: "Max file size is 5MB"; Type error: "Only JPEG/PNG formats are allowed"; Government ID: "Please upload a valid document (JPG, PNG, PDF)"; Display inline; Allow retry

**Required Field Errors**: Email: "Please enter a valid email address"; Name: "Name must be at least 2 characters"; Department: "Department is required"; Host: "Please select a host"; Purpose: "Purpose must be at least 5 characters"

### Staff Search Error Handling

**Scenarios**: Network failure: "Unable to search hosts. Please try again."; Empty results: "No hosts found matching your search"; Search too slow: Show loading spinner

**Behavior**: Display error inline below search input; Allow user to modify query and retry; Maintain selected host if error occurs; Show loading state during fetch

## 9. Accessibility

### Focus Management
**Step Transitions**: Auto-focus first input field on step render; Clear focus when navigating between steps; Trap focus in modals (if used)

**Form Focus**: `autoFocus` attribute on first input; Return focus to input after dropdown interactions; Keyboard navigation between form fields

**Back Navigation**: Focus "Back" button when arriving at step; Maintain logical tab order

### Keyboard Navigation

**Form Navigation**: Tab/Shift+Tab between fields; Enter to submit valid forms; Escape to cancel or go back; Arrow keys for dropdowns/chips

**Visit Type Selection**: Arrow keys to navigate cards; Enter/Space to select; Visual focus indicators

**Platform Chips** (DeliveryDetailsStep): Arrow keys to navigate chips; Enter/Space to select; `role="radio"` and `aria-checked` attributes

### Screen Reader Announcements

**Step Changes**: `aria-live="polite"` announces step change; "Step 3 of 6 • Delivery" on step load

**Form Validation**: `aria-describedby` links errors to fields; `aria-invalid="true"` on invalid fields; Error messages announced when focus moves to field

**Success States**: "Details Saved!" announced with `role="status"` and `aria-live="polite"`

**Staff Search**: Dropdown announced with `role="listbox"`; Search results as options; "5 hosts found" when results load

### ARIA Attributes

**Form Fields**: `aria-required="true"` for required fields; `aria-invalid` for invalid state; `aria-describedby` for error messages

**File Inputs**: `aria-label="Capture visitor photo"`; `aria-label="Upload government ID"`

**Dropdowns**: `aria-expanded` for open/close state; `aria-controls` for dropdown content; `aria-haspopup` for dropdowns

**Platform Chips**: `role="radio"` for chip group; `aria-checked` for selected state; `aria-label` for each chip
